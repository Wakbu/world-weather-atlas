# World Weather Atlas 프로그램 설명서

## 1. 개요

World Weather Atlas는 별도의 백엔드 없이 실행되는 정적 날씨 웹 애플리케이션입니다. 사용자가 도시를 검색하거나 지도에서 위치를 선택하면 브라우저가 Open-Meteo API를 호출하고 결과를 화면의 그래프와 지표로 변환합니다.

## 2. 화면 구성

### 요약

선택 위치의 현재 날씨와 생활 지표를 표시합니다.

- 현재 기온과 날씨 상태
- 체감온도, 습도, 강수량
- 풍속, 기압, 구름량
- 일출, 일몰, 낮 길이
- 자외선, 가시거리, 주간 요약
- 10분 간격 자동 갱신과 마지막 갱신 시각
- 날씨 조건에 따른 우산, 선크림, 옷차림, 수분 보충 조언
- 현재 비가 내릴 때만 표시되는 비구름 레이더 지도

### 위치 선택

- 도시명 검색 결과 목록
- 브라우저 현재 위치
- 지도 클릭 및 드래그 가능한 핀
- 주요 도시 빠른 선택

### 48시간

현재 시각부터 최대 48시간의 시간별 데이터를 표시합니다.

- 온도 선 그래프
- 강수확률 막대
- 최고·최저 기온
- 최대 강수확률과 풍속
- 3시간 간격 날씨 상태

### 주간

오늘 이후 7일간의 최고·최저 기온과 예상 강수량을 그래프로 표시합니다.

### 과거

사용자가 지정한 시작일과 종료일의 과거 날씨를 조회합니다.

- 최근 7일/30일 빠른 선택
- 최대 31일 범위 제한
- 최고·최저 기온 그래프
- 실제 강수량과 최대 풍속 요약

## 3. 데이터 흐름

```text
사용자 위치 선택
  → 위치 객체 생성
  → Open-Meteo API 요청
  → 응답 데이터 저장
  → 현재/시간별/주간/과거 렌더링 함수 실행
  → DOM 및 SVG 그래프 갱신
```

애플리케이션 상태는 `script.js`의 `state` 객체에 저장합니다.

- `selectedPlace`: 현재 선택 위치
- `weather`: 예보 및 과거 데이터
- `unit`: 섭씨 또는 화씨
- `mapPoint`: 지도에서 선택한 좌표
- `loading`: 중복 요청 방지 상태

## 4. 외부 서비스

### Open-Meteo Geocoding API

도시 검색어를 위도, 경도, 국가, 행정구역, 시간대로 변환합니다.

```text
https://geocoding-api.open-meteo.com/v1/search
```

### Open-Meteo Forecast API

현재 날씨, 시간별 데이터, 일별 예보를 한 번에 요청합니다.

```text
https://api.open-meteo.com/v1/forecast
```

### Open-Meteo Historical Weather API

날짜 범위에 해당하는 과거 일별 데이터를 요청합니다.

```text
https://archive-api.open-meteo.com/v1/archive
```

### Leaflet 및 OpenStreetMap

Leaflet이 지도 화면, 클릭 이벤트, 드래그 가능한 마커를 처리합니다. 실제 지도 타일은 OpenStreetMap에서 제공합니다.


### RainViewer Weather Maps API

현재 기상 코드가 비, 이슬비, 소나기 또는 뇌우일 때 최신 레이더 프레임을 가져옵니다. 비가 오지 않으면 레이더 영역을 숨기고 API도 호출하지 않습니다.

엔드포인트: https://api.rainviewer.com/public/weather-maps.json
## 5. 주요 구현 방식

### 그래프

외부 차트 라이브러리를 사용하지 않고 JavaScript로 SVG 요소를 생성합니다. 온도는 꺾은선, 강수량과 강수확률은 막대로 표현합니다.

### 단위 변환

API 응답은 섭씨 기준으로 저장하고 화면을 렌더링할 때 화씨로 변환합니다. 따라서 단위 변경 시 API를 다시 호출하지 않습니다.

### 다크 모드

`data-theme` 속성과 CSS 변수를 사용합니다. 처음 방문하면 시스템 설정을 따르고, 사용자가 선택한 테마는 `localStorage`에 저장합니다.


### 자동 갱신

선택 위치의 현재 날씨와 예보는 10분마다 갱신합니다. 브라우저 탭이 숨겨져 있을 때는 요청을 건너뛰고, 다시 활성화됐을 때 마지막 갱신 후 10분이 지났다면 즉시 갱신합니다. 사용자가 선택한 과거 날짜 데이터는 자동 갱신 시 유지합니다.
### 반응형 레이아웃

1120px, 820px, 560px, 380px 구간을 기준으로 패널과 지표 배치를 변경합니다. 넓은 그래프와 시간대 목록은 모바일에서 가로 스크롤로 탐색할 수 있습니다.

## 6. 파일 역할

| 파일 | 역할 |
| --- | --- |
| `index.html` | 화면 구조, 탭, 입력 폼, 템플릿 |
| `styles.css` | 레이아웃, 반응형 디자인, 다크 모드 |
| `script.js` | API 요청, 상태 관리, 지도 및 그래프 렌더링 |
| `.github/workflows/pages.yml` | GitHub Pages 자동 배포 |

## 7. 실행 및 운영 시 주의사항

- 모든 데이터 요청은 브라우저에서 직접 발생합니다.
- API나 지도 서버가 차단되면 해당 기능을 사용할 수 없습니다.
- OpenStreetMap 저작자 표시를 제거하지 않아야 합니다.
- 대규모 상업 트래픽에서는 각 외부 서비스의 이용 정책과 호출 제한을 별도로 검토해야 합니다.
- 과거 데이터는 실시간 관측소 원본이 아니라 Open-Meteo가 제공하는 역사 데이터셋을 사용합니다.

## 8. 확장 아이디어

- 위치 즐겨찾기와 최근 검색 저장
- 여러 도시 날씨 비교
- 대기질과 미세먼지 데이터
- 강수 레이더 또는 기상 레이어
- PWA 및 오프라인 기본 화면
- 백엔드 캐시를 통한 외부 API 호출 최적화

## 9. 서비스 기능 확장

### 저장과 공유

- 선택 위치, 활성 탭을 URL의 `lat`, `lon`, `name`, `tab` 매개변수에 저장합니다.
- 즐겨찾기 6곳과 최근 위치 5곳을 `localStorage`에 저장합니다.
- 즐겨찾기 중 최대 3곳의 최신 현재 날씨를 동시에 비교할 수 있습니다.
- 마지막 정상 날씨 응답은 위치별로 최대 7일간 저장하며 API 장애 시 저장 시각과 함께 표시합니다.

### 대기질과 생활 알림

Open-Meteo Air Quality API에서 미국 AQI, PM2.5, PM10, 오존, 이산화질소를 가져옵니다. 대기질 자료는 CAMS ENSEMBLE 기반이며 화면에 제공자를 표시합니다. 시간별 기상 코드, 강수확률, 예상 강수량을 함께 확인해 12시간 이내 강수 가능성이 있으면 사용자가 허용한 브라우저 알림을 보냅니다.

### 표시 설정과 접근성

- 섭씨/화씨, km/h·m/s·mph, mm·inch, hPa·inHg 단위를 선택할 수 있습니다.
- 주요 탐색 요소와 동적 날씨 설명은 한국어와 영어로 전환됩니다.
- 그래프의 데이터 점에는 네이티브 툴팁과 현재/오늘/최신 기준선을 제공합니다.
- 지도 휠 확대를 비활성화해 페이지 스크롤이 지도에 갇히지 않게 했고 모바일 선택 패널은 지도 하단에 고정됩니다.

### PWA와 오프라인

`manifest.webmanifest`가 설치 정보를 제공하고 `service-worker.js`가 앱 셸, 스타일, 스크립트, 생활 조언 이미지를 캐시합니다. 네트워크 연결이 끊기면 앱 셸과 마지막으로 저장한 날씨를 우선 표시합니다. 날씨 API 자체는 네트워크가 복구된 뒤 다시 갱신됩니다.

## 10. 추가 외부 서비스

### Open-Meteo Air Quality API

```text
https://air-quality-api.open-meteo.com/v1/air-quality
```

현재 AQI와 주요 오염물질 농도, 최대 3일 시간별 대기질 데이터를 제공합니다. Open-Meteo 및 CAMS ENSEMBLE 저작자 표시를 유지해야 합니다.
## 11. 초기화와 기상 지도

### 초기화 정책

URL에 `lat`와 `lon`이 없으면 최근 위치를 자동 적용하지 않고 기본 검색어 `Seoul`로 시작합니다. 최근 위치는 위치 선택 탭에만 남습니다. 상단 초기화 버튼은 오작동 방지를 위해 두 번 눌러야 실행되며 다음 데이터를 제거합니다.

- 즐겨찾기와 비교 도시
- 최근 조회 위치
- 마지막 정상 날씨 캐시
- 테마, 언어, 표시 단위, 강수 알림 설정
- URL의 위치 및 활성 탭 매개변수

### 기상 지도 레이어

| 레이어 | 표현 방식 | 데이터 |
| --- | --- | --- |
| 비구름 | 최신 레이더 타일 | RainViewer |
| 구름량 | 현재 화면을 채우는 6×5 연속 색상 격자 | Open-Meteo Forecast API |
| 바람 | 풍속별 색상의 고대비 방향 화살표와 수치 배지 | Open-Meteo Forecast API |
| PM2.5 | 현재 화면을 채우는 농도별 연속 색상 격자 | Open-Meteo Air Quality API |

구름·바람·PM2.5는 관측 타일이 아니라 Open-Meteo의 최신 모델 현재값을 격자로 시각화합니다. 사용자가 지도를 이동하면 500ms 후 새 화면 범위의 30개 좌표를 한 번의 다중 좌표 요청으로 다시 조회합니다. 비구름 레이어는 선택 위치의 현재 강수 여부와 관계없이 기상 지도 탭에서 확인할 수 있으며, 요약 화면의 레이더는 기존처럼 현재 비가 확인될 때만 표시됩니다.
